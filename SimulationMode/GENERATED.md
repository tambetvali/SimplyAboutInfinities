# Voyage to Laegna

The **Voyage** category gathers simulations that introduce the *spirit* of Laegna before one enters the more technical simulation panel. These are not the main task‑driven simulations; instead, they serve as **orientation portals** — visual, geometric, and experiential ways to “arrive” into Laegna’s conceptual space.  
They are placed here because the *SimulationMode* folder is the starting point for all simulation‑related materials: the task, the data, and the introductory experiences that help the reader understand the environment in which the main simulation operates.

## [Infinite Voyage](https://github.com/tambetvali/Landscopes/blob/main/Landscapes/infinitevoyage.md)

A two‑part experiential entry into Laegna. It frames the idea that infinity is not a number but a **movement**, a **relation**, and a **landscape** that can be navigated.

### [Primary Vision Portal](https://laegna-spirit-vision.lovable.app/)
A visual‑symbolic gateway into Laegna’s conceptual world.  
This simulation focuses on **perception**, **orientation**, and **symbolic resonance** — the “first contact” with Laegna’s structure. It introduces the user to the idea that infinities behave like **flows**, **octaves**, and **structural recurrences**, not like large quantities.

### [Infinities Runtime Surface (SI‑RV1V)](https://laegna-infinities-si-rv1v.bolt.host/)
A more dynamic and surface‑based exploration.  
Where the Vision Portal is symbolic, SI‑RV1V is **mechanical and spatial**: it shows how infinity behaves when treated as a **runtime surface**, a **moving relation**, and a **self‑consistent field**.  
This simulation prepares the user for the main task by showing how infinite structures stabilize, repeat, and transform.

# Main Simulation Panel

This section contains the **core simulations** directly answering the task defined in  
[Simulation Definition](https://github.com/tambetvali/SimplyAboutInfinities/blob/main/SimulationMode/README.md#simulation-definition).

These three simulations represent **three different interpretations** of the same task.  
They are all kept here — in the same folder as the task and the data — because this folder is the **development root**: the place where the problem is defined, explored, and prototyped.  
Later, these simulations may be used in different contexts, but here they form the **canonical set** of answers to the initial problem.

## [Structural Infinity Engine](https://app-c5floyik7x8h.appmedo.com)
This is the **primary and most complete** implementation of the task.  
It focuses on the **structural mechanics** of infinity: how relations repeat, how scales fold, and how the system transitions between finite and infinite states.  
It is the closest to a “reference implementation” of the simulation definition.

## [Reactive Infinity Field](https://react-9b0k6o.onspace.build/?t=1780740106014)
A more **reactive, dynamic, and interactive** interpretation.  
This simulation emphasizes **real‑time behavior**, **field responses**, and **state transitions**.  
It is more powerful in terms of interaction and responsiveness, making it ideal for exploring how the system behaves under continuous input.

## [Minimal Infinity Surface](https://4e3ccea8-b19f-4671-9297-17bf7a67c704-production.weweb.io/)
A **minimalist** but still fully functional version.  
It strips the simulation down to its **essential mechanics**, making the underlying logic easier to observe.  
Despite its simplicity, it remains a valid and useful interpretation of the task — especially for understanding the core relations without visual complexity.
